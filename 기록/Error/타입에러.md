# 타입스크립트의 try-catch문에서의 error의 타입처리
```tsx
  const handleRegister = async (user: UserRegister) => {
    try {
      await axios.post(`${import.meta.env.VITE_BASE_URL}/auth/register`, user);
      showToast('회원가입이 완료되었습니다! 로그인해 주세요.', 'success');
      return true;
    } catch (error) {
      showToast(error.response.data.message, 'error');
      return false;
    }
  };
```
위 코드대로 작성시 error은(는) unknown 형식이라고 뜬다.
그렇다고 any로 타입설정해주긴 싫어서 customErr 로 error타입을 지정해줬다.
```tsx
  const handleRegister = async (user: UserRegister) => {
    try {
      await axios.post(`${import.meta.env.VITE_BASE_URL}/auth/register`, user);
      showToast('회원가입이 완료되었습니다! 로그인해 주세요.', 'success');
      return true;
    } catch (error) {
      const customErr = error as {response: {data: {message: string}}}
      showToast(customErr.response.data.message, 'error');
      return false;
    }
  };
```
이렇게 작성시 문제없이 타입이 지정된걸 확인할 수 있었다.
그전까지는 어떻게 하는지 몰라서 any 처리하거나 eslint-disable-next-line 등을 사용해 무시하고있었지만
error를 console.log 찍어보고 안의 데이터대로 타입 지정해주니 문제가없는걸 확인했다.
```tsx
(error as axiosErrorType)
```
최종적으로 type.ts 파일에 axiosErrorType 라고 타입을 지정해줘서 간편하게 axios들의 error 타입을 지정할 수 있게 되었다.
